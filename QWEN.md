- ПЛАН НА БУДУЩЕЕ - Рефакторинг Накладных в ДокументыОснованияПлатежа + импорт ЖО4:

1. **Рефакторинг сущности**: Преобразовать таблицу "Накладные" (Invoices) в более общую сущность "ДокументыОснованияПлатежа" (BasePaymentDocuments), не сломав существующий импорт банка.

2. **Импорт ЖО4 из TSV**: Добавить в меню "Сервис" аналог импорта Банка с выбором сопоставления полей. Формат TSV:
   - дата документа, номер документа, наименование документа
   - наименование показателя (контрагент)
   - содержание операции (вид работ/номенклатура)
   - счет дебет, счет кредит, сумма

3. **Расширенная таблица ДокументыОснованияПлатежа**:
   - дата документа (YYYY-MM-DD), номер документа
   - id наименования документа (подсправочник)
   - id контрагента (автопоиск по наименованию)
   - id договора (ручная простановка)
   - id платежа из банка (связь с банковскими платежами)
   - примечание, для сверки (bool), сверено (bool)
   - Сумма ИТОГО по документу

4. **Расшифровка документа основания**:
   - id документа основания
   - содержание операции, счет дебет, счет кредит
   - id КОСГУ (автоопределение по счету дебета)
   - сумма, примечание

5. **Форма "Документы Основания"** (аналогично форме "Банк"):
   - CRUD записей, фильтры, запросы
   - Групповые операции: автоподбор платежа из банка, простановка/очистка отметок сверки, договоров, КОСГУ

6. **Форма сверки**: Платёжное поручение → сумма → расшифровка → контрагент/договор → ДокументОснование → сумма → расшифровка по документу → содержание операции
